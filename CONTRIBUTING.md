# Contributing to Tetris: Neon Fusion

Thanks for your interest in contributing! We welcome code, docs, and design contributions.

Guidelines

1. Fork the repository and create a new branch for your feature:
   - `git checkout -b feature/my-feature`
2. Keep changes focused and small. One feature or fix per PR helps reviewers.
3. Add clear commit messages and update `CHANGELOG.md` if your change is user-visible.
4. Include screenshots or a short demo video for visual changes.
5. Run the game locally and ensure there are no console errors.

Pull Request Process

- Open a PR against `main` with a descriptive title and summary.
- Link any related issue, add testing steps, and list changed files.
- Wait for reviews and address any requested changes.

Coding Style

- Use ES6+ syntax.
- Keep functions small and modular.
- Add comments for non-obvious logic, especially math/rotation logic.

Thank you — your contribution helps make this project better for everyone!
